# 需求分析文档

## 项目背景

### 问题场景
- 日常打开浏览器会打开多个标签页
- 标签页过多后难以找到想要的页面
- 不知道哪些可以清理，哪些需要保留
- 希望快速找到自己想要的页面

### 参考项目
- `tab-out/` - 现有的标签页管理插件
  - 优点：域名分组、重复检测、稍后查看
  - 缺点：UI设计不够美观、部分功能不实用

---

## 用户需求

### 核心需求（必须）

#### 1. 域名分组
**需求描述**：按域名自动分组显示标签页

**用户场景**：
- 用户打开了 10 个 GitHub 页面、5 个 YouTube 页面、3 个 Stack Overflow 页面
- 希望按域名分组查看，快速找到目标标签页

**验收标准**：
- [x] 自动识别标签页域名
- [x] 按域名分组显示
- [x] 显示每个域名的标签页数量
- [x] 支持折叠/展开

#### 2. 域名置顶
**需求描述**：置顶常用域名，持久化存储

**用户场景**：
- 用户经常访问 GitHub、YouTube 等网站
- 希望这些域名始终显示在顶部，方便快速访问

**验收标准**：
- [x] 可以置顶任意域名
- [x] 置顶状态持久化存储（关闭页面后保留）
- [x] 打开同域名的新标签页，自动显示在置顶区域
- [x] 可以取消置顶

#### 3. 一键清理
**需求描述**：关闭整个域名分组

**用户场景**：
- 用户完成了 GitHub 上的工作
- 希望一次性关闭所有 GitHub 标签页

**验收标准**：
- [x] 可以关闭单个域名分组的所有标签页
- [x] 关闭后显示 Toast 提示
- [x] 关闭时有好看的动画效果

#### 4. 重复检测
**需求描述**：显示重复标签页，支持去重

**用户场景**：
- 用户不小心打开了多个相同的页面
- 希望快速识别并关闭重复标签页

**验收标准**：
- [x] 自动检测重复标签页（相同 URL）
- [x] 显示重复数量
- [x] 支持一键去重（保留一个，关闭其他）

#### 5. 点击跳转
**需求描述**：点击标签页跳转到对应窗口

**用户场景**：
- 用户在管理面板中看到目标标签页
- 希望点击后直接跳转到该标签页

**验收标准**：
- [x] 点击标签页标题跳转到对应标签页
- [x] 如果标签页在其他窗口，自动切换窗口
- [x] 跳转后管理面板保持打开状态

#### 6. 主题切换
**需求描述**：支持深色/浅色/跟随系统主题

**用户场景**：
- 用户喜欢深色主题
- 或者希望跟随系统主题自动切换

**验收标准**：
- [x] 支持深色主题
- [x] 支持浅色主题
- [x] 支持跟随系统主题
- [x] 主题设置持久化存储

#### 7. 窗口合并
**需求描述**：合并多个浏览器窗口

**用户场景**：
- 用户打开了多个浏览器窗口
- 希望将所有标签页合并到一个窗口

**验收标准**：
- [x] 可以合并所有窗口的标签页
- [x] 合并后显示 Toast 提示

#### 8. 好看的点击特效
**需求描述**：关闭标签页时有好看的动画效果

**用户场景**：
- 用户关闭标签页时希望有视觉反馈
- 不要音效，只要好看的动画

**验收标准**：
- [x] 关闭标签页时有淡出动画
- [x] 关闭域名分组时有卡片缩小动画
- [x] 动画流畅，不卡顿

---

### 新增功能（重要）

#### 9. 休眠建议
**需求描述**：标记长时间未访问的标签页，建议清理

**用户场景**：
- 用户打开了很多标签页，但有些已经很久没访问了
- 希望系统自动标记这些标签页，建议关闭

**验收标准**：
- [x] 自动检测超过阈值未访问的标签页
- [x] 显示休眠建议列表
- [x] 显示未访问时间
- [x] 可以一键关闭休眠标签页
- [x] 可以自定义休眠阈值（1小时/2小时/4小时/8小时/1天）

#### 10. 标签页预览
**需求描述**：鼠标悬停显示页面缩略图

**用户场景**：
- 用户通过标题无法确定是哪个页面
- 希望通过缩略图快速识别

**验收标准**：
- [ ] 鼠标悬停显示页面缩略图
- [ ] 缩略图清晰，能识别页面内容
- [ ] 可以在设置中关闭此功能

#### 11. 复制单个标签页链接
**需求描述**：复制单个标签页的链接

**用户场景**：
- 用户想要分享某个标签页给其他人
- 或者保存某个标签页的链接

**验收标准**：
- [x] 鼠标悬浮在标签页上显示复制按钮
- [x] 点击复制按钮复制该标签页的链接
- [x] 复制后显示成功提示

#### 12. 搜索过滤
**需求描述**：搜索标签页标题和 URL

**用户场景**：
- 用户想要快速找到某个标签页
- 通过关键词搜索过滤

**验收标准**：
- [x] 实时搜索过滤
- [x] 支持搜索标题和 URL
- [x] 搜索时高亮显示匹配项
- [x] 按 / 快捷键聚焦搜索框
- [x] 按 Escape 清除搜索

#### 13. 设置页面
**需求描述**：高级设置集中管理

**用户场景**：
- 用户想要自定义插件行为
- 在设置页面中统一管理

**验收标准**：
- [x] 独立的设置页面
- [x] 主题设置
- [x] 显示设置（列数、缩略图）
- [x] 休眠设置（阈值、是否显示）
- [x] 置顶域名管理
- [x] 重置设置功能

## 用户场景

### 场景 1：日常浏览
**用户**：普通用户
**场景**：日常浏览网页，打开多个标签页
**需求**：快速找到目标标签页，清理不需要的标签页
**功能**：域名分组、点击跳转、一键清理

### 场景 2：开发工作
**用户**：开发者
**场景**：开发时打开大量 GitHub、Stack Overflow 页面
**需求**：快速切换不同项目页面，清理重复标签页
**功能**：域名置顶、重复检测、休眠建议

### 场景 3：研究学习
**用户**：学生/研究者
**场景**：研究时打开大量参考页面
**需求**：保存重要页面，清理过期页面
**功能**：导出功能、休眠建议、搜索过滤
